# creative-portfolio-starter

[English Version](./README.md)

这是我自己想做作品集落地页时会优先拿来用的一套结构。比起那种满屏卡片、像后台一样的模板，我更想让首页更像海报、专题页或者带明确视觉方向的项目入口。

它本来就是围绕我偏好的内容来搭的：视觉表达、动态设计、AI 辅助生产、3D，以及那些需要一个强首屏来建立气质的项目。

## 包含内容

- 一个全屏铺开的首屏结构，强调视觉层级
- 适合少量强项目的案例区，而不是泛泛的卡片墙
- 按设计、动效、3D、AI 系统分组的能力行
- 不依赖框架的轻量级入场动画
- 适合静态部署、改起来也直接的页面结构

## 文件说明

- `index.html` 页面结构
- `styles.css` 版式、视觉方向、动效和响应式规则
- `data.js` 项目数据和能力分组内容
- `main.js` 轻量渲染逻辑与动效触发

## 我为什么保留这个起始版本

很多作品集模板都在鼓励“多”，但我更在意的是：

- 首屏先给人一个明确印象
- 项目不需要多，但每个都要有存在感
- 既能看出审美，也能看出制作深度
- 联系方式清晰，但页面本身不要像销售页

## 使用方式

1. 替换 `data.js` 里的示例内容
2. 根据你自己的方向重写首屏文案
3. 把 CTA 链接换成你的联系方式或域名
4. 部署到 GitHub Pages、Netlify 或任意静态平台

## 比较值得改的地方

- 把占位案例换成你自己的真实项目
- 每个项目尽量保留一个结果、一个过程点、一个强视觉
- 工具尽量按产出分组，不要只是堆软件名
- 第一屏保持干净，别把所有信息都挤进去
