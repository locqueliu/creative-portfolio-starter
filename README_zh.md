# creative-portfolio-starter

[English Version](./README.md)

这个仓库是我自己做作品集页面时会优先拿来改的一套起始结构。

我一直不太喜欢那种一打开就一排排卡片、像后台系统一样的作品集模板。对我来说，作品集更像是一个入口页，应该先让人感受到你的气质，再慢慢往下看你做过什么。所以这个版本会更偏海报感、专题页感，也更适合放少量但更有存在感的项目。

## 里面有些什么

- 一个铺满首屏的开场结构，先把视觉气氛立住
- 一个更适合放强项目的案例区，而不是普通卡片墙
- 按设计、动效、3D、AI 系统分组的能力区
- 不靠框架、比较轻的入场动画
- 静态部署友好的页面结构

## 文件说明

- `index.html` 页面结构
- `styles.css` 版式、视觉和响应式规则
- `data.js` 项目数据和能力分组
- `main.js` 渲染逻辑和一些轻量交互

## 我为什么会一直留着这个版本

因为很多作品集模板都太“平均”了，看完很难记住人。

我更想要的是这种感觉：

- 第一屏就有明确印象
- 项目不需要很多，但每个都要站得住
- 不只是看结果，也能隐约看出你怎么做事
- 联系方式要清楚，但页面本身别像在硬推销

## 怎么用

1. 先把 `data.js` 里的示例内容换掉
2. 按你自己的方向重写首屏文案
3. 把 CTA 链接换成你的联系方式或域名
4. 部署到 GitHub Pages、Netlify 或任意静态平台

## 我觉得比较值得改的地方

- 用你自己最能打的项目替掉占位案例
- 每个项目尽量留一个结果、一个过程点、一个强视觉
- 工具别只是堆名字，最好能按产出方式分组
- 第一屏尽量克制，不要什么都想往里塞
